Unplugin Turbo Console is a build-time plugin that makes `console` statements dramatically more useful during development. Instead of hunting through your source files to find where a log came from, the plugin automatically annotates each statement with its filename, line number, and variable names — and adds a clickable link that opens the exact location in your editor. It runs at build time via [unplugin](https://unplugin.unjs.io), which means it works with every major bundler without requiring any runtime dependency.

## How it works

Unplugin Turbo Console hooks into your bundler's transform pipeline using [unplugin](https://unplugin.unjs.io). At build time, it parses each source file with an OXC-based AST parser, locates every `console.*` call, and rewrites it to include additional arguments — the source filename, line number, and the names of any variables being logged.

For the Launch Editor feature, the plugin also starts a lightweight local HTTP server (default: `127.0.0.1:3070`) when your dev server runs. The server receives requests from clickable log URLs in the browser and calls `launch-editor` to open your editor at the exact file and line. No browser extension or runtime package is needed; the transformation happens entirely at build time.

Because all modifications are applied before the code reaches the browser or Node.js runtime, Unplugin Turbo Console has zero effect on your production bundle — all enhancements are stripped automatically when `NODE_ENV` is set to `production`.

## Supported frameworks

Unplugin Turbo Console supports every major bundler and meta-framework through a single package with per-bundler entry points:

| Category | Frameworks |
|---|---|
| **Build tools** | Vite, Webpack, Rspack, Rollup, esbuild, Farm |
| **Meta-frameworks** | Nuxt, SvelteKit, Astro, Next.js, SolidStart |

All entry points are exported from the `unplugin-turbo-console` package (e.g. `unplugin-turbo-console/vite`, `unplugin-turbo-console/webpack`, `unplugin-turbo-console/nuxt`). See [Get started](/getting-started) for framework-specific setup code.

## Requirements

- **Node.js** >= 20
- One of the following bundlers (all listed as optional peer dependencies):

| Peer dependency | Required version |
|---|---|
| `vite` | >= 3 |
| `webpack` | >= 5 |
| `rollup` | >= 3 |
| `esbuild` | any |
| `@farmfe/core` | >= 1 |
| `astro` | >= 3 |
| `@nuxt/kit` | >= 3 |
| `@nuxt/schema` | >= 3 |
